# Aether Splicer — Desktop wrapper

Native desktop wrapper for `hub-bus-panel/splicer.html`. Path C of the
splicer-deployment decision: keep the existing single-file widget as the
single source of truth, but give it a dedicated OS window, system tray
presence, OS-keychain credential storage, and native notifications.

## Layout

| File | Purpose |
|---|---|
| `main.js` | Electron main process: BrowserWindow, tray, single-instance lock, login-item, safeStorage IPC handlers. |
| `preload.js` | contextBridge — exposes `window.coworkSecret` (get/set/clear) and `window.coworkNotify(title, body)` to the renderer. |
| `splicer.html` | **Local copy** of `hub-bus-panel/splicer.html`, modified to prefer `window.coworkSecret` for the bearer and to fire native notifications for `@zack` envelopes. The original browser file is **untouched**. |
| `notify.js` | Optional renderer helper — currently only requests notification permission eagerly. Kept for future renderer-side patches. |
| `gen-icon.cjs` | Generates `tray-icon.png` (32x32 placeholder square). Runs on `npm install` via `postinstall`. |
| `tray-icon.png` | Tray + window icon. Generated; **placeholder, swap when convenient**. |
| `package.json` | electron + electron-builder devDeps, NSIS build config (Windows-only). |

## Splicer.html — copy, not symlink

We chose to **copy** `hub-bus-panel/splicer.html` into this directory rather
than load it via `file://` from the sibling directory. Reasons:

1. The desktop copy needs different behavior (safeStorage hook, native
   notifications, full-window layout instead of fixed 380×600). Keeping it
   as an independent file avoids polluting the browser version with
   Electron-only branches.
2. `electron-builder` packages files listed under `build.files`. A
   `..\hub-bus-panel\splicer.html` reference would have to either copy at
   build time or be carried as an asar extra-resource — both add fragility.
3. The browser version of `splicer.html` is unchanged and still works in a
   plain browser via `start/open-splicer.bat`.

When you change the original `hub-bus-panel/splicer.html`, **manually port
the change** to this copy (or run a small `node ./tools/sync-splicer.mjs`
script — not yet written; out of scope for this build).

## Build & install (one-time)

```powershell
cd C:\Users\Falki\shunt-final-v\hub-bus-panel-desktop
npm install
```

`npm install` runs `gen-icon.cjs` via `postinstall`, so `tray-icon.png` will
be in place when Electron starts. If you ever want to regenerate it,
`npm run icon`.

## Run in dev

```powershell
cd C:\Users\Falki\shunt-final-v\hub-bus-panel-desktop
npm start
```

Or from the project root:

```powershell
npm run splicer:desktop
```

Or double-click `start\run-splicer-desktop.bat`.

## Build the installer

```powershell
cd C:\Users\Falki\shunt-final-v\hub-bus-panel-desktop
npm run build
```

Produces `dist\Aether Splicer Setup <version>.exe`. Run that to install per-user
(no admin required).

## Notes & known limits

* **No code-signing.** Windows SmartScreen will warn the first time you run
  the unsigned installer. Click "More info" → "Run anyway". The signed
  binary path is a future task — needs an EV or OV cert and a CSR through
  a CA. Not configured here.
* **Placeholder tray icon.** `tray-icon.png` is a flat blue square with a
  light-blue border, generated by `gen-icon.cjs`. Replace with real artwork
  any time; keep the same path/filename so `main.js` and `electron-builder`
  pick it up.
* **Windows-only build target.** No macOS DMG, no Linux AppImage. Adding
  them is one `target` block per platform inside `package.json` →
  `build`, but is intentionally deferred.
* **safeStorage availability.** On Windows, `safeStorage` uses DPAPI under
  the user's account. If `safeStorage.isEncryptionAvailable()` returns
  false (rare; e.g. some sandboxed contexts), the renderer transparently
  falls back to localStorage with the same yellow warning banner the
  browser version shows.
* **Only @zack notifications.** The renderer only fires a desktop toast
  for envelopes whose `to === "@zack"` and only when the window is not
  focused. Adjust the predicate in `splicer.html`'s `maybeNotify(env)` if
  you want broader coverage.
* **Single-instance.** Launching the app a second time focuses the
  existing window instead of opening a duplicate.
* **Minimize to tray.** Closing the window hides it; the app keeps
  running. Use the tray-icon's "Quit" menu item to actually exit.

## Migration from the browser splicer

If you've been using `hub-bus-panel/splicer.html` in a browser, your bearer
secret lives in `localStorage` for that browser profile. The desktop copy
starts with an empty config — paste the bearer once, and from then on it
lives in Windows Credential Vault under the
`com.halkive.aether-splicer` AppUserModelID's user-data folder
(`%APPDATA%/Aether Splicer/splicer-secret.bin`, encrypted via DPAPI).

To wipe the stored secret:

```javascript
// In DevTools (Ctrl+Shift+I in dev mode):
await window.coworkSecret.clear();
```
